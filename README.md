### Глосарій "Graph Visualizer"

- Основні технології та бібліотеки:
  - C++20: Використовується для забезпечення безпеки пам'яті (Smart Pointers) та гнучкості типів через шаблони (Templates).
  - Qt6: Кросплатформний фреймворк. Використовується для побудови GUI, обробки подій та графічного рендерингу.
  - Graphviz: Зовнішня бібліотека для автоматичного розташування вершин та експорту графів у форматах PNG, SVG, DOT, PDF, JSON, JPEG.
  - Doxygen: Інструмент для автоматичної генерації технічної документації безпосередньо з коментарів у вихідному коді.

- Предметна галузь (Теорія графів):
  - Граф: Сукупність вершин та ребер. У проєкті реалізовано через абстрактний клас Core::Graph.
  - Список суміжності: Структура, де кожна вершина зберігає список своїх сусідів.
  - Матриця суміжності: квадратна матриця, яка відображає наявність (або вагу) ребер між вершинами графа.
  - Обхід графа: систематичне відвідування вершин із використанням алгоритмів BFS (пошук у ширину), DFS (пошук у глибину) та алгоритму Дейкстри для знаходження найкоротших шляхів у зваженому графі.

- Архітектурні компоненти:
  - AlgorithmController: Керуючий компонент візуалізації. Керує життєвим циклом алгоритму, зберігає історію станів та дозволяє крокувати алгоритмом назад/вперед.
  - Strategy: Патерн, що дозволяє динамічно змінювати алгоритм обробки (BFS/DFS/Dijkstra) через спільний інтерфейс Algorithm.
  - Observer: Патерн для зв'язку логіки з GUI. Коли в графі додається вершина, він сповіщає MainWindow через інтерфейс GraphObserver.
  - RAII: Використання std::unique_ptr для автоматичного керування життєвим циклом вершин графа, запобігання витоків пам'яті.

- Візуалізація та UI (Qt)
  - VertexItem: Графічне представлення вершини. Відповідає за малювання кола, підпису та зміну кольору залежно від стану (Visited, Current, Frontier).
  - EdgeItem: Графічне представлення ребра. Динамічно оновлює свою позицію (trackVertices) при переміщенні вершин.
  - GraphScene: Спеціалізована сцена (QGraphicsScene), що забезпечує інтерактивну взаємодію: перетягування об'єктів та створення ребер.
  - AlgoState: Містить інформацію про те, які вершини та ребра були активними на конкретному кроці алгоритму.

- Тестування та GitHub Actions
  - GitHub Actions: CI/CD сервіс, що автоматично збирає проєкт та оновлює документацію при кожному оновленні коду.
  - GitHub Pages: Платформа для хостингу згенерованої Doxygen-документації, доступна за статичним посиланням.
  - Unit Testing: Автоматизовані тести для перевірки математичної коректності алгоритмів без запуску графічного інтерфейсу.
